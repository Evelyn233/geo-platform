# WordPress自动发布功能使用指南

## 🚀 功能概述

现在系统支持将生成的文章自动发布到WordPress网站的指定分类中，无需手动复制粘贴！

## 📋 前置准备

### 1. WordPress应用密码设置

在WordPress后台设置应用密码：

1. 登录WordPress后台
2. 进入 **用户 → 个人资料**
3. 滚动到 **应用密码** 部分
4. 输入应用名称（如：GEO系统）
5. 点击 **添加新应用密码**
6. 复制生成的密码（格式：`xxxx xxxx xxxx xxxx`）

### 2. 确保WordPress REST API启用

大多数WordPress网站默认启用REST API，如果遇到问题：

1. 检查是否有插件禁用了REST API
2. 确保WordPress版本 ≥ 4.7
3. 测试访问：`https://your-site.com/wp-json/wp/v2/`

## 🎯 使用方法

### 1. 填写WordPress信息

在GEO系统界面中填写：

- **WordPress URL**: 你的网站地址（如：https://example.com）
- **用户名**: WordPress用户名
- **应用密码**: 刚才生成的应用密码

### 2. 配置发布选项

- **发布分类**: 默认为"新闻资讯"，可自定义
- **自动发布**: 勾选后直接发布，不勾选则只生成不发布
- **文章数量**: 设置要生成的文章数量
- **模型选择**: 选择AI模型（推荐使用豆包Seed 1.6 Flash）

### 3. 执行自动发布

点击 **🚀 自动发布** 按钮，系统将：

1. 测试WordPress连接
2. 生成指定数量的文章
3. 自动创建分类（如果不存在）
4. 批量发布文章到WordPress
5. 显示发布结果和文章链接

## 📊 功能特性

### 自动分类管理
- 自动创建"新闻资讯"分类（如果不存在）
- 支持自定义分类名称
- 文章自动归类到指定分类

### 文章元数据
每篇发布的文章包含：
- 关键词信息
- 推广数据（点击量、转化率等）
- 生成时间
- 是否增强版标记

### 批量发布
- 支持一次发布多篇文章
- 自动处理发布间隔，避免请求过快
- 详细的发布结果反馈

### 错误处理
- 连接测试确保WordPress可访问
- 详细的错误信息提示
- 部分失败不影响其他文章发布

## 🔧 API接口

### 发布文章
```
POST /api/publish_articles
```

参数：
- `wp_url`: WordPress网站URL
- `wp_username`: 用户名
- `wp_password`: 应用密码
- `max_articles`: 文章数量
- `enhanced`: 是否使用增强模式
- `model`: AI模型
- `category_name`: 分类名称
- `auto_publish`: 是否自动发布

### 获取分类
```
GET /api/wordpress_categories
```

### 获取已发布文章
```
GET /api/published_articles
```

## 🎨 文章格式

发布的文章包含：

1. **标题**: SEO优化的标题
2. **摘要**: 文章简介
3. **内容**: 完整的文章内容
4. **标签**: 基于关键词的标签
5. **分类**: 指定的分类
6. **元数据**: 推广数据和技术信息

## ⚠️ 注意事项

### 权限要求
- 确保用户有发布文章的权限
- 应用密码权限与用户权限相同

### 性能考虑
- 建议单次发布不超过10篇文章
- 系统自动添加2秒间隔避免请求过快
- 大量发布建议分批进行

### 内容质量
- 建议使用增强模式获得更好内容
- 发布前可先测试生成效果
- 定期检查发布的内容质量

## 🐛 常见问题

### 连接失败
- 检查WordPress URL是否正确
- 确认应用密码是否有效
- 检查网络连接

### 发布失败
- 确认用户有发布权限
- 检查分类名称是否合法
- 查看详细错误信息

### 内容问题
- 检查AI模型是否正常
- 确认推广数据是否加载
- 尝试重新生成

## 📈 最佳实践

1. **测试先行**: 先用1篇文章测试发布流程
2. **分类规划**: 提前规划好文章分类结构
3. **内容审核**: 定期检查自动发布的内容质量
4. **数据备份**: 重要内容建议手动备份
5. **SEO优化**: 利用推广数据优化内容策略
